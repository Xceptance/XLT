## Purpose

Displaying cached requests in the Result Browser frontend with visual distinction, disk space optimization, and correct export to JSON/HAR formats.

## Requirements

### Requirement: Display Cached Status
The Result Browser SHALL visually distinguish requests that were served from the browser cache from standard network requests.

#### Scenario: Viewing a cached request in the list
- **WHEN** the user views the request list in the Result Browser
- **AND** a request has the `cached` flag set to `true`
- **THEN** the request row is styled differently (e.g., using a `.contentTypeCached` CSS class)
- **AND** the request size or status indicator shows "from cache"

### Requirement: Disk Space Optimization
The `DumpMgr` SHALL NOT dump duplicate response bodies for cached requests to prevent massive disk space bloat on load generators.

#### Scenario: Dumping a cached request
- **WHEN** the `DumpMgr` processes a request where `cached = true`
- **THEN** the JSON `RequestInfo` is generated
- **AND** the response payload is NOT written to the `responses/` directory

### Requirement: Graceful UI Handling of Missing Dumps
The Result Browser UI SHALL gracefully handle viewing cached requests whose bodies were not dumped.

#### Scenario: Viewing request content of a cached request
- **WHEN** the user clicks on a cached request in the Result Browser
- **THEN** the UI does not attempt to fetch the missing response file via AJAX
- **AND** the UI displays a message indicating "Response body not stored for cached requests"

### Requirement: Export Cached Status
The JSON and HAR exports generated by the Result Browser SHALL correctly reflect the cached status of requests.

#### Scenario: Exporting to JSON
- **WHEN** the `DumpMgr` generates the `data.js` file
- **THEN** the serialized `RequestInfo` objects include the `cached` property

#### Scenario: Exporting to HAR
- **WHEN** the `DumpMgr` generates the `data.har` file
- **AND** a request was served from the cache
- **THEN** the HAR entry reflects that the response came from the cache
